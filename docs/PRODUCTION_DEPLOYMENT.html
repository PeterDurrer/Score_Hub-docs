Score_Hub Production Deployment Guide ​

Dieses Dokument beschreibt das vollständige Deployment einer Score_Hub Instanz auf Azure.

Übersicht ​

Score_Hub verwendet folgende Azure-Ressourcen:

  • Azure Static Web App - Frontend (React) + API (Azure Functions)
  • Azure SQL Database - Datenbank
  • Azure Key Vault - Secrets Management (optional)
  • Application Insights - Monitoring

Voraussetzungen ​

Lokale Tools ​

powershell
# Azure CLI installieren
winget install Microsoft.AzureCLI

# Node.js 20+ installieren
winget install OpenJS.NodeJS.LTS

# SQL Server PowerShell Modul

Deployment Schritte ​

1. Automatisches Deployment (empfohlen) ​

powershell
# Konfiguration anpassen
cp scripts/deployment-config.example.ps1 scripts/deployment-config.ps1
# Werte in deployment-config.ps1 anpassen

# Deployment ausführen
.\scripts\deploy-production.ps1

2. Manuelles Deployment ​

2.1 Resource Group erstellen ​

powershell
$env = "production"
$location = "westeurope"
$rgName = "scorehub-$env-rg"

az group create --name $rgName --location $location

2.2 SQL Server & Database erstellen ​

powershell
$sqlServerName = "scorehub-$env-sql-$(Get-Random -Maximum 999999)"
$sqlDbName = "scorehub-$env-db"
$sqlAdminUser = "scorehubadmin"
$sqlAdminPassword = "IhrSicheresPasswort123!"

# SQL Server
az sql server create `
  --name $sqlServerName `
  --resource-group $rgName `
  --location $location `
  --admin-user $sqlAdminUser `
  --admin-password $sqlAdminPassword

# Firewall für Azure Services
az sql server firewall-rule create `
  --resource-group $rgName `
  --server $sqlServerName `
  --name AllowAzureServices `
  --start-ip-address 0.0.0.0 `
  --end-ip-address 0.0.0.0

# Database
az sql db create `
  --resource-group $rgName `
  --server $sqlServerName `
  --name $sqlDbName `
  --edition Basic `
  --capacity 5

2.3 Static Web App erstellen ​

powershell
$swaName = "scorehub-$env-web"

az staticwebapp create `
  --name $swaName `
  --resource-group $rgName `
  --location $location `
  --sku Free

2.4 App Settings konfigurieren ​

powershell
$jwtSecret = "IhrJWTSecretMindestens32Zeichen!"
$sqlConnectionString = "Server=tcp:$sqlServerName.database.windows.net,1433;Initial Catalog=$sqlDbName;Persist Security Info=False;User ID=$sqlAdminUser;Password=$sqlAdminPassword;MultipleActiveResultSets=False;Encrypt=True;TrustServerCertificate=False;Connection Timeout=30;"

az staticwebapp appsettings set `
  --name $swaName `
  --resource-group $rgName `
  --setting-names `
    "JWT_SECRET=$jwtSecret" `
    "SQL_CONNECTION_STRING=$sqlConnectionString"

2.5 Datenbank-Schema anwenden ​

powershell
Invoke-Sqlcmd `
  -ServerInstance "$sqlServerName.database.windows.net" `
  -Database $sqlDbName `
  -Username $sqlAdminUser `
  -Password $sqlAdminPassword `
  -InputFile "infra/database-schema.sql"

2.6 Admin-Benutzer erstellen ​

powershell
# Passwort hashen (lokal mit Node.js)
$adminPassword = "AdminPasswort123!"
$hashedPassword = node -e "console.log(require('bcryptjs').hashSync('$adminPassword', 12))"

# Admin erstellen
$adminId = [guid]::NewGuid().ToString()
$adminEmail = "admin@example.com"
$adminFirstName = "Admin"
$adminLastName = "User"

$createAdminSql = @"
-- Admin User erstellen
INSERT INTO Users (Id, Email, PasswordHash, FirstName, LastName, IsActive, CreatedAt)
VALUES ('$adminId', '$adminEmail', '$hashedPassword', '$adminFirstName', '$adminLastName', 1, GETUTCDATE());

-- Admin Rolle zuweisen
INSERT INTO UserRoles (UserId, RoleId)
SELECT '$adminId', Id FROM Roles WHERE Name = 'admin';
"@

Invoke-Sqlcmd `
  -ServerInstance "$sqlServerName.database.windows.net" `
  -Database $sqlDbName `
  -Username $sqlAdminUser `
  -Password $sqlAdminPassword `
  -Query $createAdminSql

2.7 GitHub Actions Deployment Token ​

powershell
# Deployment Token holen
$deploymentToken = az staticwebapp secrets list `
  --name $swaName `
  --resource-group $rgName `
  --query "properties.apiKey" -o tsv

Write-Host "GitHub Secret 'AZURE_STATIC_WEB_APPS_API_TOKEN_PRODUCTION' setzen auf:"
Write-Host $deploymentToken

Bekannte Probleme & Lösungen ​

Problem: npm ci schlägt fehl ​

Ursache: Keine package-lock.json vorhanden Lösung: npm install statt npm ci in CI/CD verwenden

Problem: API 404 Errors ​

Ursache: Azure Functions nicht registriert Lösung: Alle Functions müssen app.http() aufrufen und in index.ts importiert werden

Problem: API 500 bei Datenbankzugriff ​

Ursache: Connection String Format Lösung: Connection String muss als einzelner String in Azure App Settings sein (nicht JSON)

Problem: Node.js EBADENGINE Warnings ​

Ursache: Azure SDK benötigt Node 20+ Lösung: Node 20 in CI/CD und staticwebapp.config.json konfigurieren:

json
{
  "platform": {
    "apiRuntime": "node:20"
  }
}

Problem: JWT Token ungültig ​

Ursache: JWT_SECRET nicht korrekt gesetzt oder unterschiedlich zwischen Login und Verify Lösung:

  1. Prüfen dass JWT_SECRET in Azure App Settings gesetzt ist
  2. Beide auth.ts und utils/auth.ts verwenden dieselbe Variable
  3. Token nach Secret-Änderung neu generieren (logout/login)

Problem: Login funktioniert, aber /api/auth/me gibt 401 ​

Ursache: Unterschiedliche JWT_SECRET Werte oder Cold Start Timing Lösung: Lokales Debugging um das genaue Problem zu identifizieren

CI/CD Pipeline ​

Die GitHub Actions Pipeline ist in .github/workflows/ci-cd.yml konfiguriert:

  • Push to main → Deployment zu Staging
  • GitHub Release → Deployment zu Production
