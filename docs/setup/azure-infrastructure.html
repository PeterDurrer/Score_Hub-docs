Azure Infrastruktur ​

Diese Anleitung beschreibt das Deployment der Azure Ressourcen mit Bicep.

Schnellstart ​

1. Variablen setzen ​

powershell
# Konfiguration
$ENVIRONMENT = "staging"  # oder "production"
$LOCATION = "westeurope"
$PROJECT_NAME = "scorehub"
$RESOURCE_GROUP = "$PROJECT_NAME-$ENVIRONMENT-rg"

# SQL Admin Passwort (sicher speichern!)
$SQL_PASSWORD = Read-Host -AsSecureString "SQL Admin Passwort"
$SQL_PASSWORD_PLAIN = [Runtime.InteropServices.Marshal]::PtrToStringAuto(
    [Runtime.InteropServices.Marshal]::SecureStringToBSTR($SQL_PASSWORD)
)

2. Resource Group erstellen ​

powershell
az group create `
    --name $RESOURCE_GROUP `
    --location $LOCATION `
    --tags environment=$ENVIRONMENT project=ScoreHub

3. Bicep Deployment ​

powershell
az deployment group create `
    --resource-group $RESOURCE_GROUP `
    --template-file infra/main.bicep `
    --parameters `
        environment=$ENVIRONMENT `
        sqlAdminPassword=$SQL_PASSWORD_PLAIN `
        b2cTenantName="scorehub" `
        b2cClientId="your-client-id"

4. Outputs abrufen ​

powershell
$outputs = az deployment group show `
    --resource-group $RESOURCE_GROUP `
    --name main `
    --query properties.outputs | ConvertFrom-Json

Write-Host "Function App URL: $($outputs.functionAppUrl.value)"
Write-Host "Static Web App URL: $($outputs.staticWebAppUrl.value)"
Write-Host "SQL Server: $($outputs.sqlServerFqdn.value)"
Write-Host "Key Vault: $($outputs.keyVaultName.value)"

Ressourcen im Detail ​

Static Web App (Frontend) ​

Das Frontend wird als Azure Static Web App gehostet:

  • Free Tier für Staging
  • Standard Tier für Production (Custom Domain, SLA)
  • Globale Verteilung via CDN
  • Automatisches HTTPS
bicep
resource staticWebApp 'Microsoft.Web/staticSites@2023-01-01' = {
  name: '${projectName}-${environment}-web'
  location: location
  sku: {
    name: environment == 'production' ? 'Standard' : 'Free'
    tier: environment == 'production' ? 'Standard' : 'Free'
  }
}

Function App (Backend API) ​

Das Backend läuft als Azure Functions:

  • Consumption Plan: Automatische Skalierung, Pay-per-Use
  • Node.js 18 Runtime
  • Application Insights Integration
bicep
resource functionApp 'Microsoft.Web/sites@2023-01-01' = {
  name: '${projectName}-${environment}-api'
  kind: 'functionapp'
  properties: {
    serverFarmId: appServicePlan.id
    siteConfig: {
      nodeVersion: '~18'
      cors: {
        allowedOrigins: ['https://${staticWebApp.properties.defaultHostname}']
      }
    }
  }
}

SQL Database ​

Azure SQL Database für die Datenspeicherung:

  • Basic Tier für Staging (5 DTU)
  • Standard S1 für Production (20 DTU)
  • Geo-Redundantes Backup (Production)
  • Transparent Data Encryption
bicep
resource sqlDatabase 'Microsoft.Sql/servers/databases@2023-05-01-preview' = {
  name: '${projectName}-${environment}-db'
  sku: {
    name: environment == 'production' ? 'S1' : 'Basic'
    tier: environment == 'production' ? 'Standard' : 'Basic'
  }
}

Key Vault ​

Secrets Management:

  • SQL Connection String
  • Azure AD B2C Secrets
  • API Keys
powershell
# Secret hinzufügen
az keyvault secret set `
    --vault-name "$PROJECT_NAME-$ENVIRONMENT-kv" `
    --name "SqlConnectionString" `
    --value "Server=..."

Firewall Konfiguration ​

SQL Server Firewall ​

powershell
# Azure Services erlauben
az sql server firewall-rule create `
    --resource-group $RESOURCE_GROUP `
    --server "$PROJECT_NAME-$ENVIRONMENT-sql" `
    --name AllowAzureServices `
    --start-ip-address 0.0.0.0 `
    --end-ip-address 0.0.0.0

# Ihre IP für lokale Entwicklung
$myIp = (Invoke-RestMethod http://ipinfo.io/json).ip
az sql server firewall-rule create `
    --resource-group $RESOURCE_GROUP `
    --server "$PROJECT_NAME-$ENVIRONMENT-sql" `
    --name MyIP `
    --start-ip-address $myIp `
    --end-ip-address $myIp

Manuelle Schritte ​

Nach dem automatischen Deployment:

1. Static Web App API Token ​

powershell
$apiToken = az staticwebapp secrets list `
    --name "$PROJECT_NAME-$ENVIRONMENT-web" `
    --query properties.apiKey -o tsv

# Speichern Sie diesen Token für GitHub Secrets
Write-Host "API Token: $apiToken"

2. Function App Publish Profile ​

powershell
az functionapp deployment list-publishing-profiles `
    --name "$PROJECT_NAME-$ENVIRONMENT-api" `
    --resource-group $RESOURCE_GROUP `
    --xml > publish-profile.xml

Troubleshooting ​

Deployment Fehler ​

powershell
# Deployment Status prüfen
az deployment group list `
    --resource-group $RESOURCE_GROUP `
    --output table

# Fehlerdetails
az deployment group show `
    --resource-group $RESOURCE_GROUP `
    --name main `
    --query properties.error

Resource Provider Fehler ​

powershell
# Provider Status prüfen
az provider show --namespace Microsoft.Web --query registrationState

# Bei "NotRegistered":
az provider register --namespace Microsoft.Web --wait
