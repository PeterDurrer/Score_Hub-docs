Schema anwenden ​

Via Azure Portal ​

  1. Azure Portal → SQL Databases → Ihre Datenbank
  2. "Query editor (preview)"
  3. Login mit SQL Admin Credentials
  4. Kopieren Sie den Inhalt von infra/database-schema.sql
  5. "Run"

Via PowerShell ​

powershell
# Variablen
$SQL_SERVER = "scorehub-staging-sql.database.windows.net"
$SQL_DATABASE = "scorehub-staging-db"
$SQL_USER = "scorehubadmin"
$SQL_PASSWORD = Read-Host -AsSecureString "SQL Password"

# Schema anwenden
Invoke-Sqlcmd `
    -ServerInstance $SQL_SERVER `
    -Database $SQL_DATABASE `
    -Username $SQL_USER `
    -Password ([Runtime.InteropServices.Marshal]::PtrToStringAuto(
        [Runtime.InteropServices.Marshal]::SecureStringToBSTR($SQL_PASSWORD)
    )) `
    -InputFile "infra/database-schema.sql" `
    -TrustServerCertificate

Via sqlcmd ​

powershell
sqlcmd `
    -S "scorehub-staging-sql.database.windows.net" `
    -d "scorehub-staging-db" `
    -U "scorehubadmin" `
    -P "YourPassword" `
    -i "infra/database-schema.sql" `
    -C

Tabellen ​

Users ​

Speichert Benutzerinformationen aus Azure AD B2C:

sql
CREATE TABLE Users (
    Id UNIQUEIDENTIFIER PRIMARY KEY DEFAULT NEWID(),
    ExternalId NVARCHAR(255) NOT NULL UNIQUE,  -- B2C Object ID
    Email NVARCHAR(255) NOT NULL UNIQUE,
    FirstName NVARCHAR(100),
    LastName NVARCHAR(100),
    ClubId UNIQUEIDENTIFIER REFERENCES Clubs(Id),
    IsActive BIT NOT NULL DEFAULT 1,
    CreatedAt DATETIME2 NOT NULL DEFAULT GETUTCDATE(),
    lastModifiedAt DATETIME2 NOT NULL DEFAULT GETUTCDATE()
);

Results ​

Speichert Einzelschüsse und berechnete Werte:

sql
CREATE TABLE Results (
    Id UNIQUEIDENTIFIER PRIMARY KEY,
    CompetitionId UNIQUEIDENTIFIER NOT NULL,
    ShooterId UNIQUEIDENTIFIER NOT NULL,
    RangeId UNIQUEIDENTIFIER,
    GroupMemberId UNIQUEIDENTIFIER,

    -- Einzelschüsse (JSON Array)
    Shots NVARCHAR(MAX),  -- ["10","9","10",...]

    -- Berechnete Werte
    TotalScore DECIMAL(10,2),
    Tiefschuss INT,  -- Niedrigster Schuss
    SeriesScores NVARCHAR(MAX),  -- Serien-Summen

    -- Metadaten
    EnteredBy UNIQUEIDENTIFIER REFERENCES Users(Id),
    VerifiedBy UNIQUEIDENTIFIER REFERENCES Users(Id),
    CreatedAt DATETIME2 DEFAULT GETUTCDATE()
);

Migrationen ​

Für zukünftige Schema-Änderungen:

Migration erstellen ​

powershell
# Migrations-Ordner
New-Item -ItemType Directory -Path "infra/migrations" -Force

# Neue Migration
$timestamp = Get-Date -Format "yyyyMMddHHmmss"
$migrationFile = "infra/migrations/${timestamp}_description.sql"

@"
-- Migration: description
-- Date: $(Get-Date -Format "yyyy-MM-dd")

-- Up
ALTER TABLE ...;

-- Down (für Rollback)
-- ALTER TABLE ...;
"@ | Out-File $migrationFile

Write-Host "Migration erstellt: $migrationFile"

Migration anwenden ​

powershell
# Alle neuen Migrationen anwenden
Get-ChildItem "infra/migrations/*.sql" | ForEach-Object {
    Write-Host "Applying: $($_.Name)"
    Invoke-Sqlcmd -InputFile $_.FullName ...
}

Backup & Restore ​

Manuelles Backup ​

powershell
# Export als BACPAC
az sql db export `
    --resource-group "scorehub-production-rg" `
    --server "scorehub-production-sql" `
    --name "scorehub-production-db" `
    --admin-user "scorehubadmin" `
    --admin-password "YourPassword" `
    --storage-key-type "StorageAccessKey" `
    --storage-key "YourStorageKey" `
    --storage-uri "https://yourstorage.blob.core.windows.net/backups/backup.bacpac"

Monitoring ​

Query Performance ​

sql
-- Lange laufende Queries
SELECT TOP 10
    qs.total_elapsed_time / qs.execution_count AS avg_time_ms,
    qs.execution_count,
    SUBSTRING(st.text, 1, 100) AS query_text
FROM sys.dm_exec_query_stats qs
CROSS APPLY sys.dm_exec_sql_text(qs.sql_handle) st
ORDER BY avg_time_ms DESC;

Index-Nutzung ​

sql
-- Ungenutzte Indizes
SELECT
    OBJECT_NAME(i.object_id) AS TableName,
    i.name AS IndexName,
    i.type_desc,
    s.user_seeks, s.user_scans, s.user_lookups, s.user_updates
FROM sys.indexes i
LEFT JOIN sys.dm_db_index_usage_stats s ON i.object_id = s.object_id AND i.index_id = s.index_id
WHERE OBJECTPROPERTY(i.object_id, 'IsUserTable') = 1;
