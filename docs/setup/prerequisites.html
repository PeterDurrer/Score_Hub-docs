Voraussetzungen ​

Benötigte Software ​

Node.js 18+ ​

powershell
# Installation via winget
winget install OpenJS.NodeJS.LTS

# Überprüfung
node --version  # sollte v18.x.x oder höher sein
npm --version

Azure CLI ​

powershell
# Installation via winget
winget install Microsoft.AzureCLI

# Login
az login

# Subscription setzen
az account set --subscription "Ihre-Subscription-ID"

Azure Functions Core Tools ​

powershell
# Installation via npm
npm install -g azure-functions-core-tools@4 --unsafe-perm true

# Überprüfung
func --version  # sollte 4.x sein

Git ​

powershell
# Installation via winget
winget install Git.Git

# Konfiguration
git config --global user.name "Ihr Name"
git config --global user.email "ihre.email@example.com"

Visual Studio Code ​

powershell
# Installation via winget
winget install Microsoft.VisualStudioCode

# Empfohlene Extensions werden automatisch vorgeschlagen

Azure Ressourcen ​

Azure Subscription ​

Sie benötigen eine aktive Azure Subscription. Falls Sie noch keine haben:

  1. Erstellen Sie ein Azure-Konto
  2. Für Neukunden: CHF 200 Startguthaben

Benötigte Berechtigungen ​

  • Subscription: Contributor Rolle
  • Azure AD: Berechtigung zum Erstellen von App Registrations

Resource Provider registrieren ​

powershell
# Notwendige Provider aktivieren
az provider register --namespace Microsoft.Web
az provider register --namespace Microsoft.Sql
az provider register --namespace Microsoft.KeyVault
az provider register --namespace Microsoft.Storage
az provider register --namespace Microsoft.Insights
az provider register --namespace Microsoft.AzureActiveDirectory

GitHub ​

Repository erstellen ​

  1. Erstellen Sie ein neues Repository auf GitHub
  2. Aktivieren Sie GitHub Actions
  3. Erstellen Sie die Environments staging und production

GitHub CLI (optional) ​

powershell
# Installation
winget install GitHub.cli

# Login
gh auth login

Lokale Entwicklung ​

SQL Server (optional) ​

Für lokale Entwicklung mit echten Daten:

powershell
# SQL Server Express installieren
winget install Microsoft.SQLServer.2022.Express

# SQL Server Management Studio
winget install Microsoft.SQLServerManagementStudio

Alternativ: Azure SQL für lokale Entwicklung verwenden (Firewall-Regel erforderlich).

Überprüfung ​

Führen Sie dieses Script aus, um Ihre Installation zu überprüfen:

powershell
Write-Host "Überprüfe Installationen..." -ForegroundColor Cyan

$checks = @(
    @{ Name = "Node.js"; Command = "node --version"; Required = "v18" },
    @{ Name = "npm"; Command = "npm --version"; Required = "9" },
    @{ Name = "Azure CLI"; Command = "az version --output tsv"; Required = "2" },
    @{ Name = "Func Tools"; Command = "func --version"; Required = "4" },
    @{ Name = "Git"; Command = "git --version"; Required = "2" }
)

foreach ($check in $checks) {
    try {
        $result = Invoke-Expression $check.Command 2>$null
        if ($result -match $check.Required) {
            Write-Host "✓ $($check.Name): $result" -ForegroundColor Green
        } else {
            Write-Host "⚠ $($check.Name): $result (erwartet: $($check.Required))" -ForegroundColor Yellow
        }
    } catch {
        Write-Host "✗ $($check.Name): nicht gefunden" -ForegroundColor Red
    }
}

Write-Host "`nÜberprüfung abgeschlossen." -ForegroundColor Cyan
