Code Konventionen ​

TypeScript ​

  • Strict Mode aktiviert
  • Keine any Types (eslint Regel)
  • Interfaces über Types bevorzugen
  • Funktionale Komponenten mit Hooks

Naming ​

ElementKonventionBeispiel
KomponentenPascalCaseUserProfile.tsx
HookscamelCase mit useuseAuth.ts
UtilitiescamelCaseformatDate.ts
ConstantsSCREAMING_SNAKEAPI_BASE_URL
CSS Classeskebab-caseuser-profile

Git Workflow ​

  1. Feature Branch von develop erstellen
  2. Commits mit Conventional Commits
  3. Pull Request erstellen
  4. Review & Merge zu develop
  5. develop → main für Release

Conventional Commits ​

feat: Neues Feature
fix: Bugfix
docs: Dokumentation
style: Formatierung
refactor: Code Refactoring
test: Tests
chore: Build/Config Änderungen
