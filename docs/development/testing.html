Testing ​

Diese Seite beschreibt die Teststrategie und -implementierung für ScoreHub.

Test Pyramide ​

ScoreHub folgt der klassischen Testpyramide mit Fokus auf Unit Tests:

     ┌─────────────┐  E2E Tests (langsam, teuer)
     │   End-to-End │  └─ Browser Tests
     │    Tests     │     └─ Kritische User Journeys
     └─────────────┘
            │
     ┌─────────────┐  Integration Tests (mittel)
     │ Integration │  └─ API Endpunkt Tests
     │    Tests    │     └─ Datenbank Integration
     └─────────────┘
            │
     ┌─────────────┐  Unit Tests (schnell, günstig)
     │   Unit      │  └─ Funktions-Tests
     │   Tests     │     └─ Komponenten-Tests
     └─────────────┘

Unit Tests ​

Frontend Komponenten ​

typescript
// Beispiel: Button Komponente Test
import { render, screen, fireEvent } from "@testing-library/react";
import { Button } from "@/components/Button";

describe("Button", () => {
  it("renders with correct text", () => {
    render(<Button>Click me</Button>);
    expect(screen.getByText("Click me")).toBeInTheDocument();
  });

  it("calls onClick when clicked", () => {
    const handleClick = vi.fn();
    render(<Button onClick={handleClick}>Click me</Button>);

    fireEvent.click(screen.getByText("Click me"));
    expect(handleClick).toHaveBeenCalledTimes(1);
  });
});

Backend Funktionen ​

typescript
// Beispiel: Auth Funktion Test
import { describe, it, expect, vi } from "vitest";
import { validateToken } from "../src/utils/auth";

describe("validateToken", () => {
  it("returns user for valid token", async () => {
    const mockToken = "valid.jwt.token";
    const result = await validateToken(mockToken);

    expect(result).toEqual({
      id: "user-123",
      email: "user@example.com",
    });
  });

  it("throws error for invalid token", async () => {
    const mockToken = "invalid.token";

    await expect(validateToken(mockToken)).rejects.toThrow("Invalid token");
  });
});

Integration Tests ​

API Endpunkt Tests ​

typescript
// Beispiel: Clubs API Test
import request from "supertest";
import { app } from "../src/functions";

describe("Clubs API", () => {
  it("GET /api/clubs returns list of clubs", async () => {
    const response = await request(app).get("/api/clubs").expect(200);

    expect(Array.isArray(response.body)).toBe(true);
    expect(response.body[0]).toHaveProperty("id");
    expect(response.body[0]).toHaveProperty("name");
  });

  it("POST /api/clubs creates new club", async () => {
    const newClub = {
      name: "Test Club",
      address: "Test Address",
    };

    const response = await request(app)
      .post("/api/clubs")
      .send(newClub)
      .expect(201);

    expect(response.body).toHaveProperty("id");
    expect(response.body.name).toBe(newClub.name);
  });
});

End-to-End Tests ​

Playwright Setup ​

typescript
// Beispiel: E2E Test mit Playwright
import { test, expect } from "@playwright/test";

test("user can login and view dashboard", async ({ page }) => {
  // Gehe zur Login Seite
  await page.goto("/login");

  // Fülle Login Formular aus
  await page.fill('[data-testid="email"]', "user@example.com");
  await page.fill('[data-testid="password"]', "password123");

  // Klicke Login Button
  await page.click('[data-testid="login-button"]');

  // Überprüfe Dashboard
  await expect(page).toHaveURL("/dashboard");
  await expect(page.locator("h1")).toContainText("Dashboard");
});

Mocking & Test Data ​

API Mocking ​

typescript
// MSW für API Mocking
import { rest } from "msw";
import { setupServer } from "msw/node";

const server = setupServer(
  rest.get("/api/clubs", (req, res, ctx) => {
    return res(
      ctx.json([
        { id: "1", name: "Club A" },
        { id: "2", name: "Club B" },
      ])
    );
  })
);

beforeAll(() => server.listen());
afterEach(() => server.resetHandlers());
afterAll(() => server.close());

CI/CD Integration ​

GitHub Actions Test Workflow ​

yaml
name: Tests
on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: "18"

      - name: Install dependencies
        run: npm install

      - name: Run frontend tests
        run: cd frontend && npm test -- --coverage

      - name: Run backend tests
        run: cd api && npm test -- --coverage

      - name: Upload coverage
        uses: codecov/codecov-action@v3

Best Practices ​

Test Naming ​

typescript
// ✅ Gute Test Namen
describe("UserProfile Component", () => {
  it("displays user name when data is loaded", () => {});
  it("shows loading spinner while fetching data", () => {});
  it("displays error message when request fails", () => {});
});

// ❌ Schlechte Test Namen
describe("UserProfile", () => {
  it("works", () => {});
  it("test 1", () => {});
  it("should render", () => {});
});

Test Isolation ​

  • Jeder Test ist unabhängig
  • Keine Seiteneffekte zwischen Tests
  • Datenbank wird vor jedem Test zurückgesetzt

Test First Development ​

  1. Test schreiben (rot)
  2. Code implementieren (grün)
  3. Code refaktorieren (refactor)

Debugging Tests ​

Häufige Probleme ​

  1. Async Code: Immer await verwenden
  2. DOM Updates: act() für React State Updates
  3. Mock Cleanup: Mocks vor jedem Test zurücksetzen
