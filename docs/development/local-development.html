Lokale Entwicklung ​

Diese Anleitung zeigt, wie Sie ScoreHub lokal entwickeln und debuggen.

Schnellstart ​

1. Repository klonen ​

powershell
git clone https://github.com/your-org/scorehub.git
cd scorehub

2. Dependencies installieren ​

powershell
# Root (npm workspaces)
npm install

# Oder einzeln
cd frontend && npm install
cd ../backend && npm install

3. Konfiguration ​

Backend (backend/local.settings.json) ​

Erstellen Sie die Datei aus dem Template:

powershell
Copy-Item backend/local.settings.json.example backend/local.settings.json

Passen Sie die Werte an:

json
{
  "IsEncrypted": false,
  "Values": {
    "FUNCTIONS_WORKER_RUNTIME": "node",
    "AzureWebJobsStorage": "UseDevelopmentStorage=true",
    "SQL_CONNECTION_STRING": "Server=localhost;Database=ScoreHub;User Id=sa;Password=YourPassword;TrustServerCertificate=true",
    "B2C_TENANT_NAME": "scorehubauth",
    "B2C_CLIENT_ID": "your-client-id",
    "B2C_POLICY_NAME": "B2C_1_SignUpSignIn"
  },
  "Host": {
    "CORS": "*",
    "CORSCredentials": false
  }
}

Frontend (frontend/.env.local) ​

env
VITE_API_BASE_URL=http://localhost:7071/api
VITE_MSAL_CLIENT_ID=your-client-id
VITE_MSAL_AUTHORITY=https://scorehubauth.b2clogin.com/scorehubauth.onmicrosoft.com/B2C_1_SignUpSignIn
VITE_MSAL_KNOWN_AUTHORITY=https://scorehubauth.b2clogin.com

4. Starten ​

Option A: F5 in VS Code (empfohlen) ​

  1. Öffnen Sie VS Code im Projektroot
  2. Drücken Sie F5 oder wählen Sie "Full Stack" im Debug Panel
  3. Frontend startet auf http://localhost:5173
  4. Backend startet auf http://localhost:7071

Option B: Manuell ​

powershell
# Terminal 1: Backend
cd backend
npm run start

# Terminal 2: Frontend
cd frontend
npm run dev

VS Code Debug-Konfiguration ​

Die .vscode/launch.json enthält vorkonfigurierte Debug-Profile:

Full Stack (Compound) ​

Startet Frontend und Backend gleichzeitig:

json
{
  "name": "Full Stack",
  "configurations": ["Frontend: Vite", "Backend: Azure Functions"]
}

Nur Frontend ​

json
{
  "name": "Frontend: Vite",
  "type": "chrome",
  "request": "launch",
  "url": "http://localhost:5173",
  "webRoot": "${workspaceFolder}/frontend/src"
}

Nur Backend ​

json
{
  "name": "Backend: Azure Functions",
  "type": "node",
  "request": "attach",
  "port": 9229,
  "preLaunchTask": "func: host start"
}

Datenbank Setup ​

Option 1: Azure SQL (empfohlen) ​

  1. Fügen Sie Ihre IP zur Firewall hinzu:
powershell
$myIp = (Invoke-RestMethod http://ipinfo.io/json).ip
az sql server firewall-rule create `
    --resource-group "scorehub-staging-rg" `
    --server "scorehub-staging-sql" `
    --name "DevMachine-$(hostname)" `
    --start-ip-address $myIp `
    --end-ip-address $myIp
  1. Connection String in local.settings.json:
json
{
  "Values": {
    "SQL_CONNECTION_STRING": "Server=scorehub-staging-sql.database.windows.net;Database=scorehub-staging-db;User Id=scorehubadmin;Password=YourPassword;Encrypt=true"
  }
}

Option 2: Lokaler SQL Server ​

  1. SQL Server Express installieren
  2. Datenbank erstellen:
powershell
sqlcmd -S localhost -Q "CREATE DATABASE ScoreHub"
sqlcmd -S localhost -d ScoreHub -i infra/database-schema.sql
  1. Connection String:
json
{
  "Values": {
    "SQL_CONNECTION_STRING": "Server=localhost;Database=ScoreHub;Integrated Security=true;TrustServerCertificate=true"
  }
}

Option 3: Docker SQL Server ​

powershell
# SQL Server Container starten
docker run -e "ACCEPT_EULA=Y" -e "SA_PASSWORD=YourStrong@Passw0rd" `
    -p 1433:1433 --name sqlserver `
    -d mcr.microsoft.com/mssql/server:2022-latest

# Schema anwenden
docker exec -i sqlserver /opt/mssql-tools/bin/sqlcmd `
    -S localhost -U sa -P "YourStrong@Passw0rd" `
    -Q "CREATE DATABASE ScoreHub"

docker exec -i sqlserver /opt/mssql-tools/bin/sqlcmd `
    -S localhost -U sa -P "YourStrong@Passw0rd" -d ScoreHub `
    -i /path/to/database-schema.sql

Authentication Bypass (Development) ​

Für lokale Entwicklung ohne B2C können Sie die Auth temporär deaktivieren.

Backend: Mock Auth ​

Erstellen Sie backend/src/utils/auth-dev.ts:

typescript
// NUR für lokale Entwicklung!
export const mockAuth = {
  userId: "dev-user-123",
  email: "dev@localhost",
  roles: ["admin"],
  clubID: null,
};

export const getDevUser = () => mockAuth;

In der Function:

typescript
// auth.ts - nur wenn NODE_ENV === 'development'
if (
  process.env.NODE_ENV === "development" &&
  !request.headers.get("authorization")
) {
  return { user: getDevUser(), roles: ["admin"] };
}

Frontend: Mock Token ​

typescript
// config/msal-dev.ts
export const mockAccount = {
  homeAccountId: "dev",
  localAccountId: "dev-user-123",
  environment: "localhost",
  tenantId: "localhost",
  username: "dev@localhost",
  name: "Developer",
};

WARNING

Entfernen Sie alle Mock-Auth Konfigurationen vor dem Deployment!

Hot Reload ​

Frontend ​

Vite bietet automatischen Hot Module Replacement (HMR):

  • Komponenten-Änderungen: Sofortige Aktualisierung
  • CSS-Änderungen: Sofortige Aktualisierung
  • TypeScript-Fehler: Im Browser angezeigt

Backend ​

Azure Functions Core Tools unterstützen Watch Mode:

powershell
cd backend
func start --typescript --verbose

Änderungen an TypeScript-Dateien triggern automatisch einen Rebuild.

Debugging ​

Frontend Debugging ​

  1. Breakpoints in VS Code setzen
  2. F5 drücken (Chrome öffnet sich)
  3. Zur Stelle navigieren
  4. Breakpoint wird getroffen

Backend Debugging ​

  1. Breakpoints in VS Code setzen
  2. F5 drücken (Backend startet)
  3. API Request senden (z.B. via Postman)
  4. Breakpoint wird getroffen

React DevTools ​

Installieren Sie die Chrome Extension für erweiterte React-Debugging-Features:

  • Component Tree
  • Props/State Inspection
  • Profiler

Network Debugging ​

Im Browser DevTools → Network Tab:

  • API Requests überwachen
  • Response Bodies inspizieren
  • CORS Probleme identifizieren

Häufige Probleme ​

Port bereits in Verwendung ​

powershell
# Port 5173 freigeben
netstat -ano | findstr :5173
taskkill /PID <PID> /F

# Port 7071 freigeben
netstat -ano | findstr :7071
taskkill /PID <PID> /F

CORS Fehler ​

Überprüfen Sie backend/local.settings.json:

json
{
  "Host": {
    "CORS": "*"
  }
}

TypeScript Fehler ​

powershell
# Cache löschen
cd frontend && rm -rf node_modules/.vite
cd backend && rm -rf dist

# Neu installieren
npm install

Azure Functions startet nicht ​

powershell
# Azure Functions Core Tools Version prüfen
func --version

# Muss 4.x sein
npm install -g azure-functions-core-tools@4 --unsafe-perm true
