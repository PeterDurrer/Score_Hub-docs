Role Area Migration Guide ​

This guide explains how to apply the role area model changes to your Score_Hub database and application.

Overview ​

This update implements an area-based role model with three distinct scopes:

  • Global: System-wide roles (assigned to users directly)
  • Club: Club-specific roles (assigned within a club context)
  • Competition: Competition-specific roles (assigned within a competition context)

Prerequisites ​

  • SQL Server / Azure SQL Database access
  • Database backup (recommended before running migrations)
  • Admin access to the Score_Hub application

Migration Steps ​

1. Database Migrations ​

Execute the following SQL migration scripts in order:

bash
# Run migrations from infra/database/migrations/ directory

# 1. Add area column to roles table
sqlcmd -S <server> -d <database> -i add-role-area.sql

# 2. Seed area-specific roles
sqlcmd -S <server> -d <database> -i seed-area-specific-roles.sql

# 3. Create area enforcement triggers
sqlcmd -S <server> -d <database> -i create-area-enforcement-triggers.sql

# 4. Create user auto-role assignment trigger
sqlcmd -S <server> -d <database> -i create-user-auto-role-trigger.sql

# 5. Update competitionUsers table to use roleID
sqlcmd -S <server> -d <database> -i update-competitionUsers-roleID.sql

# 6. Create competition creator trigger
sqlcmd -S <server> -d <database> -i create-competition-creator-trigger.sql

2. Verify Database Changes ​

After running migrations, verify:

sql
-- Check roles have area column
SELECT roleID, name, area, description FROM roles ORDER BY area, name;

-- Check triggers exist
SELECT name, object_name(parent_id) as table_name 
FROM sys.triggers 
WHERE name LIKE 'trg_%area%' OR name LIKE 'trg_%role%' OR name LIKE 'trg_competitions%';

-- Test new user creation (should auto-assign Standard-Benutzer)
-- Test competition creation (should auto-assign creator as Wettkampfadmin)

3. Deploy API Changes ​

The API has been updated to:

  • Support area filtering for roles endpoint
  • Validate area consistency in role assignments
  • Use new German role names
  • Support roleID in competitionUsers instead of role strings
bash
# Build and deploy API
cd api
npm install
npm run build
# Deploy to Azure Functions / your hosting environment

4. Deploy Frontend Changes ​

The frontend has been updated to:

  • Include area field in Role type
  • Support area filtering in roles API calls
  • Use roleID for competition user assignments
bash
# Build and deploy frontend
cd frontend
npm install
npm run build
# Deploy to Azure Static Web Apps / your hosting environment

Testing Checklist ​

After deployment, test the following scenarios:

Database Trigger Tests ​

  • [ ] Create a new user: Verify they automatically get "Standard-Benutzer" (global) role

    sql
    -- Check userRoles after creating a user
SELECT u.email, r.name, r.area 
FROM users u 
JOIN userRoles ur ON u.userID = ur.userID 
JOIN roles r ON ur.roleID = r.roleID 
WHERE u.email = 'test@example.com';

  • [ ] Create a new competition: Verify creator gets "Wettkampfadmin" (competition) role

    sql
    -- Check competitionUsers after creating a competition
SELECT c.name, u.email, r.name, r.area
FROM competitions c
JOIN competitionUsers cu ON c.competitionID = cu.competitionID
JOIN users u ON cu.userID = u.userID
JOIN roles r ON cu.roleID = r.roleID
WHERE c.name = 'Test Competition';

  • [ ] Try assigning wrong area role: Verify it fails

    sql
    -- Should fail: trying to assign club role to userRoles
INSERT INTO userRoles (userID, roleID) 
SELECT 'some-user-id', roleID 
FROM roles 
WHERE area = 'club' AND name = 'Verein-Admin';
-- Expected: Error about area mismatch

API Tests ​

  • [ ] GET /api/roles?area=global: Returns only global roles
  • [ ] GET /api/roles?area=club: Returns only club roles
  • [ ] GET /api/roles?area=competition: Returns only competition roles
  • [ ] POST /api/user-roles with club role: Returns 400 error
  • [ ] POST /api/club-users with global role: Returns 400 error
  • [ ] POST /api/competitions/{id}/users with global role: Returns 400 error

Frontend Tests ​

  • [ ] User registration: New users can log in and have Standard-Benutzer role
  • [ ] Role management UI: Can filter roles by area
  • [ ] Club user assignment: Can only select club-area roles
  • [ ] Competition user assignment: Can only select competition-area roles
  • [ ] Menu visibility: Menus display based on user's roles

Rollback Plan ​

If you need to rollback the changes:

sql
-- 1. Drop triggers
DROP TRIGGER IF EXISTS trg_userRoles_area_check;
DROP TRIGGER IF EXISTS trg_clubUsers_area_check;
DROP TRIGGER IF EXISTS trg_competitionUsers_area_check;
DROP TRIGGER IF EXISTS trg_users_auto_assign_standard_role;
DROP TRIGGER IF EXISTS trg_competitions_assign_creator_admin;

-- 2. Drop constraints (optional - can keep area column with data)
ALTER TABLE roles DROP CONSTRAINT IF EXISTS CK_roles_area;
ALTER TABLE roles DROP CONSTRAINT IF EXISTS UQ_roles_name_area;

-- 3. Make area nullable (if you want to keep it but disable validation)
ALTER TABLE roles ALTER COLUMN area NVARCHAR(20) NULL;

Common Issues ​

Issue: "Role 'Standard-Benutzer' not found" ​

Solution: Ensure you ran the seed-area-specific-roles.sql migration script.

Issue: "Cannot assign role to user" ​

Solution: Check that the role's area matches the assignment context (global → userRoles, club → clubUsers, competition → competitionUsers).

Issue: Competition creation fails ​

Solution: Verify that:

  1. createdBy field is set when creating competitions
  2. "Wettkampfadmin" (competition) role exists
  3. trg_competitions_assign_creator_admin trigger exists

Issue: competitionUsers table still uses 'role' string column ​

Solution: Run the update-competitionUsers-roleID.sql migration. This will:

  1. Add roleID column
  2. Migrate existing data
  3. Add foreign key constraint
  4. (Optionally) drop old 'role' column
